# CDN 加速方案

## 项目现状

| 项目 | 详情 |
|------|------|
| 框架 | Next.js 16（standalone 模式） |
| 部署 | Docker 容器 + Nginx 反代（阿里云服务器） |
| 域名 | chengnanblog.cn（已备案） |

---

## 方案一：阿里云 CDN（推荐，国内最优）

### 配置步骤

1. **开通阿里云 CDN 服务**
   - 阿里云控制台 → 搜索 "CDN" → 开通
   - 计费方式选 **按流量计费**（个人博客更省钱）

2. **添加加速域名**
   - 加速域名：`chengnanblog.cn`
   - 业务类型：图片小文件
   - 源站配置：IP 回源，填阿里云服务器公网 IP + 端口 80
   - 回源协议：HTTP

3. **配置 CNAME 解析**
   - CDN 会分配一个 CNAME 地址（如 `xxx.kunlunaq.com`）
   - 去 DNS 控制台，把域名解析从 A 记录 改为 CNAME 记录

4. **配置 HTTPS**
   - CDN 控制台 → 域名管理 → HTTPS 配置 → 上传/选择 SSL 证书

5. **缓存策略**
   - `/_next/static/`（JS/CSS/图片）：缓存 30 天
   - HTML 页面：不缓存或缓存几分钟
   - `/api/` 路由：不缓存

6. **（可选）配置 `next.config.mjs` 使用独立 CDN 域名**

   如果用单独域名（如 `cdn.chengnanblog.cn`）分发静态资源：

   ```js
   const nextConfig = {
     assetPrefix: 'https://cdn.chengnanblog.cn',
     // ...其他配置
   }
   ```

   > 如果对主域名做全站加速，则不需要设 `assetPrefix`。

## ⚠️ 安全防护（必做）

按量付费 CDN 存在 **刷量攻击** 风险（恶意制造大量流量导致天价账单），以下措施 **务必配置**：

| 措施 | 操作位置 | 重要程度 |
|------|---------|---------|
| **设置带宽/流量封顶** | CDN → 用量封顶 → 设带宽上限（如 10Mbps），超出自动停止 | ⭐⭐⭐⭐⭐ 必做 |
| **设置账单预警** | 费用中心 → 每日/月费用报警（如超 ￥10 短信通知） | ⭐⭐⭐⭐⭐ 必做 |
| **IP 频次限制** | CDN → 安全配置 → 单 IP 100次/秒 | ⭐⭐⭐⭐ 推荐 |
| **购买流量包** | 提前买固定流量包（如 100GB/￥16） | ⭐⭐⭐⭐ 推荐 |
| **Referer 防盗链** | CDN → 配置 Referer 白名单 | ⭐⭐⭐ 建议 |

---

## 结论

- 个人博客正常使用 CDN 费用几乎为零（每月 ￥1 以内）
- **必须配置封顶和报警**，防止被恶意刷量
- 如果担心风险，暂不开 CDN 也完全 OK，阿里云服务器在国内，速度已经可以
- 另一个安全选择：Cloudflare 免费版，自带 DDoS 防护，不存在刷量计费问题
